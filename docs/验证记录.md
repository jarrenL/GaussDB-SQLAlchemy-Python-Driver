# GaussDB 真实环境验证记录

## 验证范围

本次验证目标是确认项目能否基于华为 `gaussdb` Python DB-API 包，通过 SQLAlchemy 方言连接真实 GaussDB 实例并执行基础 SQL。

验证内容：

- GaussDB 服务端口连通性
- 服务器端 `gsql` 基础查询
- 服务器端 Python `gaussdb` DB-API 导入和连接
- SQLAlchemy 方言注册和连接
- SQLAlchemy Core 基础 DDL、DML、查询和清理
- 文本字段返回类型

## 环境结论

实际验证环境返回的数据库版本为：

```text
GaussDB Kernel 507.0.0
```

该版本与项目最初需求中提到的轻量化集中式 505.1 不完全一致。当前修复按 GaussDB Kernel 版本格式做了通用解析，后续仍需在 505.1 环境单独回归。客户目标兼容模式已明确为 A 兼容和 B 兼容。

## 发现的问题

### 1. 本机 macOS 无法直接导入 gaussdb

现象：

```text
ImportError: no pq wrapper available
```

原因：

本机可以安装 `gaussdb` Python 包，但缺少可用的 GaussDB/libpq native client 实现。

结论：

本项目方言层是纯 Python，但底层 `gaussdb` 包依赖 native client。Windows 交付时必须验证客户端库、DLL 和 PATH。

### 2. SQLAlchemy 初始化无法解析 GaussDB 版本

现象：

SQLAlchemy PostgreSQL 基类调用 `select pg_catalog.version()` 后，按 PostgreSQL 版本格式解析，无法处理：

```text
gaussdb (GaussDB Kernel 507.0.0 ...)
```

修复：

在方言基类中覆盖 `_get_server_version_info()`，支持以下版本格式：

- `GaussDB Kernel x.y.z`
- `PostgreSQL x.y.z`
- `EnterpriseDB x.y.z`

### 3. 默认 SQL_ASCII 时文本字段返回 bytes

现象：

不传 `client_encoding` 时，真实环境会话编码为 `SQL_ASCII`，文本字段返回 `bytes`。

修复：

方言默认向底层 DB-API 传入：

```text
client_encoding=UTF8
```

如果用户确实需要其他编码，可以在连接串 query string 中覆盖。

### 4. SQLAlchemy statement cache 警告

现象：

SQLAlchemy 提示 driver 方言类未显式设置 `supports_statement_cache`。

修复：

在具体 driver 方言类 `GaussDBDialect_gaussdb` 上显式设置：

```python
supports_statement_cache = True
```

### 5. PostgreSQL 新版本系统表字段不兼容

现象：

SQLAlchemy PostgreSQL 方言会根据 `server_version_info` 选择反射查询。GaussDB 返回 `GaussDB Kernel 507.0.0`，如果直接把该版本号作为 PostgreSQL 版本号，SQLAlchemy 会访问当前 GaussDB 不存在或不兼容的系统表字段，例如：

- `pg_attribute.attgenerated`
- `pg_attribute.attidentity`
- 与 `pg_type.typcollation` 相关的相关子查询

修复：

- 将 `gaussdb_server_version_info` 保留为真实 GaussDB 内核版本
- 将 `server_version_info` 映射为保守的 PostgreSQL 兼容版本 `(9, 2)`
- 覆盖 `get_columns()` 和 `get_multi_columns()`，使用更保守的 GaussDB 兼容系统表查询支持基础列反射

## 验证结果

真实环境 SQLAlchemy 验证通过：

- 方言 `gaussdb.gaussdb` 可加载
- A 兼容库 `datcompatibility=A` 可连接
- B 兼容临时库 `datcompatibility=B` 可连接
- `gaussdb_server_version_info` 可识别为 `(507, 0, 0)`
- `server_version_info` 使用保守的 PostgreSQL 兼容版本，避免 SQLAlchemy 访问当前 GaussDB 不存在的新版本系统表列
- 默认 schema 可识别为字符串
- `show client_encoding` 返回 `UTF8`
- `select 1` 成功
- 建表成功
- 插入成功
- 查询成功
- 事务回滚成功
- 批量插入成功
- SQLAlchemy ORM CRUD 成功
- SQLAlchemy 元数据反射成功
- 连接池基础复用成功
- 文本字段返回 `str`
- 清理测试表成功
- B 兼容库中反引号、`ifnull()`、`auto_increment` 探针通过
