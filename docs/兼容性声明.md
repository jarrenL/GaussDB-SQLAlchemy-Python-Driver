# 兼容性声明

## 1. 当前支持范围

当前项目面向 GaussDB 轻量化集中式 505.1 的 A 兼容和 B 兼容场景，提供 SQLAlchemy 2.x 方言和 DB-API 适配层。

当前实现基于华为 `gaussdb` Python DB-API 包，底层仍走 GaussDB Python 驱动和对应客户端库，不是 Oracle 原生协议驱动，也不是 MySQL 原生协议驱动。

## 2. 已验证环境

已在以下真实环境完成验证：

```text
GaussDB Kernel 507.0.0
```

验证过的兼容模式：

- A 兼容库：`datcompatibility = A`
- B 兼容库：`datcompatibility = B`

注意：目标 505.1 环境仍需单独回归。

## 3. 已验证能力

A 兼容和 B 兼容均已验证：

- SQLAlchemy 方言加载
- 数据库连接
- `select 1`
- 建表、插入、查询、删表
- 事务回滚
- 批量插入
- ORM 基础 CRUD
- 列元数据反射
- 主键反射
- 唯一约束反射
- 普通索引反射
- 序列和 `nextval()` 默认值
- Alembic Operations：建表、加列、删表
- 连接池基础复用

## 4. 已识别不兼容项

### 4.1 不支持 MySQL 原生协议

当前包的连接串是：

```text
gaussdb+gaussdb://...
```

它不是：

```text
mysql+pymysql://...
mysql+mysqlconnector://...
```

因此不能声明为 MySQL 原生协议驱动。

### 4.2 不支持 Oracle 原生协议

当前包不是：

```text
oracle+oracledb://...
```

因此不能声明为 Oracle 原生协议驱动。

### 4.3 B 兼容库中 `current_timestamp()` 未通过

实测 SQL：

```sql
select current_timestamp();
```

结果：

```text
syntax error at or near ")"
```

建议：

```sql
select current_timestamp;
```

或使用目标库确认支持的时间函数写法。

### 4.4 A 兼容库中部分 MySQL 风格语法未通过

在 A 兼容库 `datcompatibility = A` 中，以下 MySQL 风格语法未通过：

```sql
select 1 as `value`;
select ifnull(null, 'fallback');
create table t (id int auto_increment primary key, name varchar(20));
```

原因：

A 兼容库是 Oracle 兼容方向，不应按 MySQL B 兼容行为预期。

### 4.5 PostgreSQL 新版本系统表字段不可直接使用

GaussDB 返回内核版本如：

```text
GaussDB Kernel 507.0.0
```

但不能直接把它当作 PostgreSQL 507 版本处理。实测以下 PostgreSQL 新版本系统表字段或查询路径不兼容：

- `pg_attribute.attgenerated`
- `pg_attribute.attidentity`
- `pg_type.typcollation` 相关查询
- `unnest(...) with ordinality`

项目已通过保守反射查询规避这些问题。

### 4.6 HSTORE 不默认支持

HSTORE 是 PostgreSQL 扩展，不假设 GaussDB A/B 兼容库默认可用。

项目默认关闭 native HSTORE。

### 4.7 Alembic autogenerate 未声明支持

当前只验证了 Alembic Operations：

- `create_table`
- `add_column`
- `drop_table`

未声明支持 Alembic autogenerate 差异检测。

## 5. 待验证项

以下能力尚未完成验证，不能作为当前版本承诺：

- Windows 真实环境安装和连接
- GaussDB 轻量化集中式 505.1 专项环境
- SSL 连接
- Oracle PL/SQL 完整兼容
- MySQL 存储过程、触发器、复杂 DDL
- MySQL M-Compatibility
- 表达式索引
- 分区表
- 视图反射
- Alembic autogenerate
- 复杂数据类型完整映射

## 6. 结论

当前版本可声明：

```text
支持 GaussDB A 兼容和 B 兼容数据库的 SQLAlchemy 基础接入能力。
```

不应声明：

```text
完整兼容 Oracle。
完整兼容 MySQL。
支持 Oracle/MySQL 原生协议。
支持 M-Compatibility。
```
