# 常见问题

## 1. 这个项目是不是从零实现 GaussDB 协议？

不是。

本项目定位是 GaussDB SQLAlchemy 方言和适配层。底层连接能力通过 pyodbc 调用 GaussDB ODBC Driver，项目重点解决 SQLAlchemy 连接串、方言注册、版本识别、事务、ORM 和反射等应用侧接入问题。

## 2. Windows 上还需要 GaussDB 客户端 DLL 吗？

不需要额外安装数据库客户端 DLL。

当前方案使用 ODBC 后端，Windows 前置条件是：

- GaussDB ODBC Driver（安装后自动注册到 ODBC 数据源管理器）
- Python 包 `pyodbc`（随本项目 wheel 自动安装）

不需要 Java，不需要 libpq。

## 3. ODBC 驱动从哪里来？

优先从以下渠道获取：

- 华为云 GaussDB 控制台的驱动或工具下载入口
- DBA 或 GaussDB 运维人员提供的交付包
- 项目客户侧统一提供的 GaussDB ODBC Driver

Windows 安装后驱动会自动注册到 ODBC 数据源管理器，可在「驱动程序」选项卡查看。

Linux 需要将 GaussDB ODBC 驱动 `.so` 路径配置到 `/etc/odbcinst.ini`，并安装 `unixODBC`。

macOS 需要通过 `brew install unixodbc` 安装 ODBC Driver Manager，再配置 GaussDB ODBC 驱动。

测试时通过连接串参数指定驱动名称：

```text
gaussdb+odbc://user:password@host:port/postgres?driver=GaussDB+ODBC+Driver&sslmode=disable
```

如果使用 DSN 模式（需先在 ODBC 数据源管理器或 `odbc.ini` 中配置 DSN）：

```text
gaussdb+odbc://user:password@/postgres?dsn=MyGaussDB
```

## 4. 如何跑真实数据库集成测试？

设置环境变量：

```bash
export GAUSSDB_TEST_URL='gaussdb+odbc://user:password@host:port/postgres?driver=GaussDB+ODBC+Driver&sslmode=disable'
pytest -m integration
```

Windows PowerShell：

```powershell
$env:GAUSSDB_TEST_URL = 'gaussdb+odbc://user:password@host:port/postgres?driver=GaussDB+ODBC+Driver&sslmode=disable'
pytest -m integration
```

集成测试会创建并删除 `gdbdrv_*_ut` 前缀的临时表。

## 5. 当前已验证哪些 SQLAlchemy 能力？

当前集成测试覆盖：

- SQLAlchemy Core 基础 DDL、DML、查询
- 事务回滚
- 批量插入
- ORM CRUD
- 元数据反射
- 常用数据类型
- 主键、唯一约束、普通索引反射
- 序列和默认值
- Alembic Operations
- Alembic autogenerate 基础无差异检测
- 复杂索引、表达式索引、视图反射和分区表反射
- 连接池基础复用

## 6. Alembic 能直接用吗？

当前已注册 `gaussdb` Alembic DDL 实现，基础 Operations 已通过真实库验证：

- 建表
- 加列
- 删表

复杂迁移和 autogenerate 仍建议在目标库上按业务模型单独回归。

## 7. 多线程并发有什么限制？

本项目底层通过 pyodbc 调用 ODBC 驱动。当前 DB-API `threadsafety = 1`：模块可以多线程共享，但连接对象不应跨线程共享。

建议：

- 每个线程通过 SQLAlchemy engine 独立获取连接。
- 不要把同一个 connection/session 在多个线程之间传递。
- 服务启动时可先做一次单线程连接预热，再进入高并发请求处理。

## 8. 当前包是否支持 A 兼容、B 兼容和 M 兼容？

当前目标已扩展为 A 兼容、B 兼容和 M 兼容。

真实环境探针显示：

- A 兼容库可执行部分 Oracle 风格语法，例如 `dual`、`nvl`、`sysdate`、`rownum`
- B 兼容库可执行部分 MySQL B 兼容语法，例如反引号、`ifnull()`、`auto_increment`
- M 兼容库可执行 MySQL 风格语法，例如反引号、`ifnull()`、`current_timestamp()`、`auto_increment`
- A/B/M 兼容库均通过了 SQLAlchemy Core、事务、ORM、反射、常用数据类型、Alembic Operations 和 autogenerate 基础探针

仍需继续验证：

- Windows 实机连接
- GaussDB 505.1 专项环境
- SSL 连接
- 客户真实模型下的 Alembic 迁移
- GaussDB 特有数据类型和系统表差异

## 9. 支持 SQLAlchemy 的 on_conflict_do_update 吗？

不支持。

GaussDB 不支持 PostgreSQL `ON CONFLICT` upsert 语法。SQLAlchemy PostgreSQL 方言的 `insert(...).on_conflict_do_update()` 会生成 `ON CONFLICT` SQL，在库上会由数据库返回语法或能力限制错误。

建议业务侧使用目标兼容模式支持的替代写法，或在应用层拆成先查后插、先更新后插等显式流程。

## 10. M 兼容下 LIKE、INTERSECT、EXCEPT 和临时表有什么差异？

M 兼容更接近 MySQL 风格，以下差异属于数据库兼容模式行为，不是驱动 bug：

- `LIKE` 默认大小写不敏感；A/B 兼容下通常为大小写敏感。
- `INTERSECT` / `EXCEPT` 不支持；SQLAlchemy 的 `intersect()` / `except_()` 在 M 兼容库上会执行失败。
- raw SQL `CREATE TEMP TABLE` 不支持，应改为 `CREATE TEMPORARY TABLE`。

如果通过 SQLAlchemy Core 创建临时表，建议显式使用 `prefixes=["TEMPORARY"]`，避免生成不兼容的临时表语法。

## 11. ODBC 驱动名称不匹配怎么办？

如果连接时报驱动未找到错误，按以下步骤排查：

- 执行 `python -c "import pyodbc; print(pyodbc.drivers())"` 查看系统已注册的全部 ODBC 驱动名称。
- 确认连接串中 `driver` 参数与列表中的名称完全一致（包括空格和大小写）。
- Windows 上注意 32 位 / 64 位匹配：64 位 Python 只能看到 64 位 ODBC 驱动，反之亦然。
- 如果驱动名称包含空格，在连接串中用 `+` 或 `%20` 替代空格，例如 `driver=GaussDB+ODBC+Driver`。

## 12. Linux 上 pyodbc 连接报错怎么办？

Linux 上需要安装 `unixODBC` 并配置 GaussDB ODBC 驱动：

```bash
# 安装 unixODBC
sudo apt install unixodbc unixodbc-dev   # Debian/Ubuntu
sudo yum install unixODBC unixODBC-devel # CentOS/RHEL

# 配置驱动（/etc/odbcinst.ini）
[GaussDB ODBC Driver]
Description = GaussDB ODBC Driver
Driver = /path/to/libgaussdb_odbc.so
```

确认驱动已注册：

```bash
odbcinst -j
odbcinst -q -d
```

## 13. macOS 上如何配置 ODBC？

```bash
# 安装 unixODBC
brew install unixodbc

# 配置驱动（~/.odbcinst.ini 或 /usr/local/etc/odbcinst.ini）
[GaussDB ODBC Driver]
Description = GaussDB ODBC Driver
Driver = /path/to/libgaussdb_odbc.so
```

macOS 上 pyodbc 默认通过 Homebrew 的 unixODBC 加载驱动，确认 `ODBCINSTINI` 环境变量指向正确的配置文件。
